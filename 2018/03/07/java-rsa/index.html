    • java-rsa

    问题

    java 环境中，在服务器端和客户端通过同一对RSA秘钥进行加解密时，加密后不能正常解密，在服务端主机加密后服务端可以解密，在客户端主机加密后客户端也可以解密，客户端加密后发送到服务端，解密失败。经过大量比对验证，确认秘钥对一致，加密解密代码一致。
    问题在于jdk不一致，在不同jdk环境中，下面代码执行的方式有区别

    在标准jdk中默认填充方式是RSA/ECB/PKCS1Padding。在android中默认填充方式是RSA/ECB/NoPadding
    导致双方不能互相解密

    解决办法

    标准jdk中
    Cipher.getInstance(“RSA”)获取Cipher对象
    非标准jdk
    Cipher.getInstance(“RSA/ECB/PKCS1Padding”) 获取对象

    同时可以解决解密后乱码问题。
